Normalization

For each date \(d\), the app normalizes against the user-selected control column from the same day. If a `DMSO` column is detected, it is preselected by default.

$$ \widetilde{I}_{\mathrm{control},d} = \mathrm{median}\left(I_{\mathrm{control},d}\right) $$ $$ \mathrm{ratio}_{i,d} = \frac{I_{i,d}}{\widetilde{I}_{\mathrm{control},d}} $$ $$ \log_2\mathrm{FC}_{i,d} = \log_2\left(\mathrm{ratio}_{i,d}\right) $$

Outliers

Outliers are detected within each `date × condition` group using the IQR rule. The app returns both:

  • `with_outliers`: all points retained
  • `without_outliers`: flagged points excluded and normalization recalculated

    Methods

    Input model

    Each CSV contains a date column and one column per treatment. Every non-empty cell is interpreted as one independent embryo.

    Normalization by date

    Within each file and each acquisition date, the selected control column is treated as the internal anchor for that day. If a `DMSO` column is detected, the interface selects it by default, but the user can choose another control when needed.

    $$ \widetilde{I}_{\mathrm{control},d} = \mathrm{median}\left(I_{\mathrm{control},d}\right) $$ $$ \mathrm{ratio}_{i,d} = \frac{I_{i,d}}{\widetilde{I}_{\mathrm{control},d}} $$ $$ \log_2\mathrm{FC}_{i,d} = \log_2\left(\mathrm{ratio}_{i,d}\right) $$

    Outlier handling

    Outliers are detected within each `date × condition` group using the 1.5×IQR rule. The app returns both the complete branch and the cleaned branch.

    $$ \mathrm{IQR} = Q_3 - Q_1 $$ $$ L_{\mathrm{inf}} = Q_1 - 1.5\cdot \mathrm{IQR}, \qquad L_{\mathrm{sup}} = Q_3 + 1.5\cdot \mathrm{IQR} $$

    Outputs

    Each uploaded CSV is processed independently. The browser app lets you inspect plots, warnings and downloadable CSV tables for each file.

    Choose control column

    Select one control column for each file before running the analysis. If a DMSO column is detected, it is preselected by default.
